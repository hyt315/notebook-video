# Dependency and asset policy

Notebook Video keeps source downloads small and reproducible by separating manifests, installed dependencies and runtime assets.

| Item | Stored in Git | Local behavior | Reason |
| --- | --- | --- | --- |
| `package.json`, `package-lock.json` | Yes | `npm ci` installs exact versions when `prepare-browser` or `render` first runs | Reproducible dependency graph |
| `node_modules` | No | Downloaded locally and reusable between renders | Generated, platform-dependent and large |
| Remotion rendering browser | No | `prepare-browser` downloads it once when needed | Platform-dependent binary |
| TTS client or model | No | Selected and connected externally by the user or agent | Provider-neutral; avoids unnecessary SDKs and credentials |
| Generated narration | No | Supplied or generated by the user before `sync` | Voice and redistribution rights vary by provider |
| Image-generation client or model | No | Selected from the current agent/platform when available | Provider-neutral; Codex enhancement is optional |
| Official generated illustration | Yes | Copied into the editable project and tracked in `visual-assets.json` | Demonstrates image-plus-text composition; source, crop, hash and rights note are recorded |
| Source Han Sans CN and Smiley Sans | Yes | Copied into each editable project | Deterministic Chinese layout; SIL OFL notices included |
| Procedural WAV effects | Yes | Copied with the template | Small deterministic runtime assets with a reuse notice |
| Renders, caches and temporary files | No | Created locally and excluded from ZIP packages | Outputs are not source dependencies |

Do not commit installed packages, downloaded browsers or unused image generations. Update JavaScript dependencies by changing the manifest and lockfile together, run `npm audit`, then run the complete validators. Remotion and any image/TTS provider remain governed by their own terms regardless of how they are connected.
